次の手順に従って作業を進めてください。
1. `gh issue view` を使用して Issue の詳細を取得する  
2. Issue に記載されている問題点を正確に理解する  
3. リポジトリ内から関連するファイルを検索する  
4. 問題を修正するために必要な変更を実装する  
5. 修正を検証するためのテストコードを作成し、実行する  
6. コードが lint チェックおよび型チェックを通過することを確認する  
7. 内容を適切に説明したコミットメッセージを作成する  
8. 変更を push し、develop ブランチへ Pull Request を作成する  

GitHub に関するすべての作業は GitHub CLI を使用して行ってください。
